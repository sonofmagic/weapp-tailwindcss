# 主持人口播卡片版

## 使用说明

这份不是逐字稿。

这是给你放在屏幕边上看的“讲述提示卡”。

每一段只保留：

- 这一段要讲什么
- 这一段最该说的 3 句话
- 这一段不要讲散的地方

适合场景：

- 你想半脱稿讲
- 你临场状态好，想更自然一点
- 你已经读过完整稿，只需要提词

---

## 卡 1：开场

这一段要讲什么：

- 先把主题打出去
- 先抛结论
- 先告诉观众这不是插件分享，是新链路

最该说的 3 句话：

1. 2026 年了，小程序样式开发不该还是一行一行手写 `margin-left: 20rpx`。
2. 今天我想讲的不是一个插件，而是一条真正能跑起来的新链路。
3. 一句话描述界面，AI 出 Tailwind 类名，`weapp-tailwindcss` 负责把它落到小程序。

不要讲散：

- 不要一上来就展开仓库细节
- 不要一上来就解释太多原理

---

## 卡 2：痛点

这一段要讲什么：

- 先让观众认同旧范式确实慢
- 把问题讲成工作流问题，不只是 CSS 问题

最该说的 3 句话：

1. 小程序样式开发慢，核心不是因为 CSS 难，而是因为工作流低效。
2. 真正慢在三件事：表达成本高、试错成本高、反馈链路长。
3. 你脑子里想的是一个界面，手上写出来的是一堆零碎属性。

不要讲散：

- 不要列太多痛点例子
- 不要在这里提前讲太多解决方案

---

## 卡 3：新范式

这一段要讲什么：

- 把三层模型讲清楚
- 让观众知道 AI、Tailwind、`weapp-tailwindcss` 分别干什么

最该说的 3 句话：

1. 新范式不是先想 CSS 怎么写，而是先想我想要什么界面。
2. 你负责表达意图，AI 负责生成 Tailwind 类名，`weapp-tailwindcss` 负责适配到小程序。
3. AI 最值钱的地方，不是一步到位，而是把“从想到到看到”这一步明显缩短。

不要讲散：

- 不要开始讲具体配置
- 不要开始讲兼容性细节

---

## 卡 4：为什么是 Tailwind

这一段要讲什么：

- 解释为什么 AI 更适合写 Tailwind，而不是直接写 CSS

最该说的 3 句话：

1. Tailwind 比传统 CSS 更适合给 AI 当输出语言。
2. Tailwind 的类名是可枚举、可组合、可约束的，AI 更容易稳定输出。
3. Tailwind 的价值不只是少写字，而是让样式变成一种可被 AI 理解的语言。

不要讲散：

- 不要变成 Tailwind 布道
- 不要讲成“Tailwind 万能”

---

## 卡 5：项目位置

这一段要讲什么：

- 把 `weapp-tailwindcss` 从“小插件”抬到“工程通路”

最该说的 3 句话：

1. `weapp-tailwindcss` 不只是一个兼容插件，它在做的是一条工程通路。
2. 它支持多种构建工具基底，也支持多种小程序开发路线。
3. 它有 demo、模板、文档、Skill、回归和 benchmark，这说明它不是 PPT 工程。

不要讲散：

- 不要逐个框架讲太细
- 不要开始念 README

---

## 卡 6：为什么现在就值得试

这一段要讲什么：

- 把这件事从“新鲜”讲成“高频刚需”

最该说的 3 句话：

1. 这套东西值得现在就试，因为它解决的是高频、重复、低价值消耗特别多的问题。
2. 如果 AI 能稳定帮你拉起卡片、列表、按钮区块这些第一稿样式，它对日常开发就是直接有价值的。
3. 我的建议不是明天全量迁移，而是你下一个页面就开始试。

不要讲散：

- 不要变成空泛的 AI 未来论
- 不要讲成“大而全迁移方案”

---

## 卡 7：Skill

这一段要讲什么：

- 讲 AI 不是自由发挥，而是按流程办事

最该说的 3 句话：

1. Skill 最重要的价值，不是让 AI 更聪明，而是让 AI 更有流程感。
2. 它会先收集上下文，再输出配置、验证步骤和回滚方案。
3. 真正能进团队流程的 AI，不是最会炫技的那个，而是最会按清单办事的那个。

不要讲散：

- 不要在这里演示过多命令
- 不要把 Skill 讲成神秘能力

---

## 卡 8：收尾

这一段要讲什么：

- 重新收回主观点
- 给观众明确回去怎么试

最该说的 3 句话：

1. 别再把小程序样式开发当成手写 CSS 的体力活了。
2. 把它变成一句可描述的意图，让 AI 用 Tailwind 说出来，再让 `weapp-tailwindcss` 帮你翻译到小程序里。
3. 回去就做三步：先跑 demo，再找一个真实模块，再用一句真实需求试一次。

不要讲散：

- 不要结尾又开新坑
- 不要把 CTA 说得太长

---

## 最后 30 秒卡

你只要记住这 4 句：

1. 今天我讲的不是插件，是一条链路。
2. Tailwind 不是为了少写字，是为了让样式变成可被 AI 理解的语言。
3. `weapp-tailwindcss` 不是小工具，它是这条链路的工程承载点。
4. 如果你回去只做一件事，就从下一个页面开始试“一句话出第一稿”。
