# Workspace Guidelines (`tools/*`)

## 适用范围

- 本文件适用于 `tools/*` 下所有私有工具项目。
- 若具体工具存在更近一级 `AGENTS.md`，以更近一级规则为准。

## 核心职责

- `tools/*` 仅承载仓库内部开发、测试、基准、生成等辅助逻辑。
- 工具项目默认 `private: true`，不作为 npm 发布包维护。

## 变更原则

- 不把运行时或对外 API 逻辑迁入工具项目。
- 工具若需要访问发布包源码，优先通过 workspace 包名或明确的仓库根路径定位，避免依赖脆弱的相对层级。
- 工具生成的临时或中间产物应留在工具项目内；只有明确属于发布包的产物才写回目标包目录。

## 测试要求

- 修改工具行为时，优先运行对应包的脚本或引用该工具的测试。
- 若工具被 CI 或 E2E 复用，需验证对应 CI/E2E 入口仍可运行。

## 推荐验证命令

- `pnpm --filter <tool-name> <script>`
- `pnpm test`
- `pnpm e2e:ci`

## 提交前检查

- 确认工具包保持 `private: true`。
- 确认没有把私有工具包加入任何发布包的 `dependencies`。
