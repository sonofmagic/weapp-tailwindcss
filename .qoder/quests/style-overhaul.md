# Website 样式全面重构设计文档

## 设计目标

对 weapp-tailwindcss 文档网站进行全方位的样式重构，打造一个兼具科技感和优秀阅读体验的现代化文档站点，确保在亮色和暗色模式下均提供一流的视觉效果。

## 核心设计原则

### 视觉层级与可读性

- 清晰的视觉层级结构，引导用户注意力流
- 优化文本对比度，确保长时间阅读不疲劳
- 合理的留白与间距，避免视觉拥挤
- 响应式排版系统，适配各种设备尺寸

### 科技感设计语言

- 现代化的渐变与光影系统
- 流畅的动效与交互反馈
- 精致的微交互细节
- 统一的设计语言贯穿全站

### 亮暗模式一致性

- 亮色模式：清新明亮、专业可靠
- 暗色模式：沉浸舒适、减少视觉疲劳
- 平滑的主题切换过渡
- 两种模式下保持品牌识别度

### 动效与过渡系统

#### 过渡时长规范

- 微交互：150ms - 200ms (按钮点击、链接悬停)
- 标准过渡：300ms - 400ms (卡片悬停、弹窗显示)
- 复杂动画：500ms - 800ms (页面切换、大型组件展开)
- 主题切换：600ms (亮暗模式切换)

#### 缓动函数

- 进入动画：cubic-bezier(0.21, 0.72, 0.28, 0.99) (ease-out-quart)
- 退出动画：cubic-bezier(0.76, 0, 0.24, 1) (ease-in-quart)
- 弹性动画：cubic-bezier(0.68, -0.55, 0.265, 1.55) (ease-out-back)
- 平滑动画：cubic-bezier(0.4, 0, 0.2, 1) (ease-in-out)

### 性能优化策略

#### CSS 优化

- 移除未使用的样式：使用 PurgeCSS 或 Tailwind CSS 内置清除
- 关键 CSS 内联：首屏样式内联到 HTML
- 非关键 CSS 延迟加载：使用 media="print" onload 技巧
- CSS 文件压缩：使用 cssnano 或等效工具

#### 字体优化

- 字体子集化：仅包含实际使用的字符
- 字体格式：WOFF2 优先，WOFF 回退
- 字体加载策略：font-display: swap
- 字体预加载：首屏字体使用 `<link rel="preload">`

#### 图片与图标优化

- 图片格式：WebP 优先，PNG/JPG 回退
- 响应式图片：使用 srcset 和 sizes
- 懒加载：非首屏图片添加 loading="lazy"
- 图标方案：Iconify 按需加载，减少打包体积

#### 动画性能

- 使用 GPU 加速属性：transform, opacity
- 避免触发重排的属性：width, height, margin, padding
- 使用 will-change 提示浏览器：仅在动画元素上
- 减少动画复杂度：移动端降低动画层级

#### 主题切换优化

- 使用 CSS 变量：避免大量样式重写
- 过渡节流：限制过渡动画数量
- 状态持久化：localStorage 缓存用户偏好
- 初始化优化：根据系统偏好预设主题，避免闪烁

## 风险与挑战

### 技术风险

- Docusaurus 主题定制限制可能影响部分样式实现
- 某些动效在低端设备上可能导致性能问题
- 主题切换动画在 Safari 浏览器可能存在兼容性问题

### 解决方案

- 通过 CSS Module 和 Swizzling 绕过 Docusaurus 限制
- 使用 matchMedia 检测设备能力，降级动画复杂度
- 针对 Safari 提供回退样式，确保功能可用

### 维护性考虑

- 建立完整的设计系统文档，便于后续维护
- 使用 CSS 变量和 Tailwind 配置集中管理样式
- 组件化设计，减少样式重复和冲突
- 定期进行视觉回归测试，确保更新不破坏现有样式
