# 从 v4 迁移到 v5

v5 的变化很直接：小程序端的 Tailwind CSS 生成交给 `WeappTailwindcss`。别再手动 patch，也别在同一个小程序构建里再挂 Tailwind 官方插件。

这背后有一个关键变化：`weapp-tailwindcss` 不再只是把已有 class 做小程序转义的“转义器”，而是转变成了“转义生成器”。它会参与 Tailwind CSS 生成链路，在同一份 Tailwind 输入上生成适合小程序环境的转义结果。

## 先改这几处

1. 升级 `weapp-tailwindcss`
2. 删除 `postinstall: "weapp-tw patch"`
3. 从小程序构建里移除 `tailwindcss`、`@tailwindcss/postcss`、`@tailwindcss/vite`
4. 注册 `WeappTailwindcss`
5. 多端项目删掉 H5/Web 上的 `disabled: weappTailwindcssDisabled`
6. 跑一次构建

## 1. 升级依赖

继续用 Tailwind CSS 3：

```bash
pnpm add -D weapp-tailwindcss@5 tailwindcss@3
```

用 Tailwind CSS 4：

```bash
pnpm add -D weapp-tailwindcss@5 tailwindcss@4
```

这些包如果只给小程序构建用，可以删：

```bash
pnpm remove tailwindcss-patch @tailwindcss/vite @tailwindcss/postcss
```

H5 / Web 构建还要用 `@tailwindcss/vite` 或 `@tailwindcss/postcss` 的话，就留着。只要别让小程序构建同时跑两套 Tailwind。

## 2. 删除 patch

删掉 `package.json` 里的这行：

```jsonc title="package.json"
{
  "scripts": {
    // 删除
    "postinstall": "weapp-tw patch"
  }
}
```

v5 不需要安装后 patch。留着只会干扰排查。

## 3. 检查 CSS 入口

Tailwind CSS 3 还是老写法：

```css title="src/app.css"
@tailwind base;
@tailwind components;
@tailwind utilities;
```

Tailwind CSS 4 用 CSS-first：

```css title="src/app.css"
@import "tailwindcss";
@source "./**/*.{vue,js,ts,jsx,tsx,wxml,axml,ttml,mpx}";
@source not "../dist";
@source not "../unpackage";
@source not "../node_modules";
```

Tailwind CSS 4 的入口放在纯 `.css` 文件里，不要直接写进 `scss`、`less`、`sass`。

## 4. 注册 WeappTailwindcss

Vite：

```ts title="vite.config.ts"
import { defineConfig } from 'vite'
import { WeappTailwindcss } from 'weapp-tailwindcss/vite'

export default defineConfig({
  plugins: [
    WeappTailwindcss({
      rem2rpx: true,
    }),
  ],
})
```

uni-app Vite 放在 `uni()` 后面：

```ts title="vite.config.ts"
plugins: [
  uni(),
  WeappTailwindcss({
    rem2rpx: true,
  }),
]
```

Webpack：

```js title="webpack.config.js"
const { WeappTailwindcss } = require('weapp-tailwindcss/webpack')

module.exports = {
  plugins: [
    new WeappTailwindcss({
      rem2rpx: true,
    }),
  ],
}
```

Taro、Mpx、uni-app Webpack 这类项目，看对应框架页的完整写法就行。迁移时重点是换成 `WeappTailwindcss`，不要再让 PostCSS 跑 Tailwind。

## 5. 多端项目删掉 H5/Web 禁用逻辑

旧版多端配置里常见这类写法：

```ts title="vite.config.ts"
const isH5 = process.env.UNI_PLATFORM === 'h5'
const weappTailwindcssDisabled = isH5

WeappTailwindcss({
  disabled: weappTailwindcssDisabled,
})
```

v5 里，`WeappTailwindcss` 可以在 H5/Web 构建中保留启用。未显式设置 `generator.target` 时，插件会根据 `UNI_PLATFORM=h5`、`UNI_UTS_PLATFORM=h5/web/web-*`、`MPX_CLI_MODE=web`、`MPX_CURRENT_TARGET_MODE=web`、`TARO_ENV=h5` 自动把生成器目标切到 `web`，输出浏览器原生 Tailwind CSS，而不是小程序转义 CSS。

迁移时可以直接改成：

```ts title="vite.config.ts"
WeappTailwindcss({
  rem2rpx: true,
})
```

如果你有明确的 Web 对照构建，也可以显式写：

```ts title="vite.config.ts"
WeappTailwindcss({
  generator: {
    target: 'web',
  },
})
```

`disabled` 仍然可以用于完全跳过插件，例如 RN、Harmony、App 这类不希望进入 `weapp-tailwindcss` 生成链路的独立构建。但对 uni-app、uni-app x、Mpx、Taro 的 H5/Web 构建，不再需要 `disabled: weappTailwindcssDisabled`。

## 6. cssEntries 不是每个项目都要写

普通 Vite 项目通常不用写。入口 CSS 被 Vite 引入后，插件会自动识别。

这些情况再手动配：

- 多个 Tailwind CSS 入口
- 入口 CSS 没有被框架引入
- Webpack、Gulp、自定义构建
- 构建日志提示找不到入口

```ts title="vite.config.ts"
import path from 'node:path'
import { WeappTailwindcss } from 'weapp-tailwindcss/vite'

WeappTailwindcss({
  cssEntries: [
    path.resolve(__dirname, 'src/app.css'),
  ],
})
```

多个入口就都写进去。

## 7. 清理 PostCSS

小程序构建里不要再放 Tailwind：

```js title="postcss.config.js"
module.exports = {
  plugins: {
    tailwindcss: {},
    '@tailwindcss/postcss': {},
  },
}
```

业务 PostCSS 插件可以留。Tailwind 生成交给 `WeappTailwindcss`。

## 8. 跑一次

```bash
pnpm build
```

或：

```bash
pnpm dev:mp-weixin
```

找个页面写一行：

```html
<view class="mt-4 rounded bg-sky-500 p-4 text-white">Hello</view>
```

能看到样式，主链路就通了。

## 出问题先看这里

| 现象 | 先查 |
| --- | --- |
| 完全没有样式 | CSS 入口有没有被构建器引入 |
| Tailwind CSS 3 类名没生成 | `tailwind.config.*` 的 `content` |
| Tailwind CSS 4 类名没生成 | CSS 入口里的 `@source` |
| 样式重复或很怪 | 小程序构建里是不是还注册了 Tailwind 官方插件 |
| 安装后还在跑 patch | `package.json` 里是不是还留着 `postinstall` |
| JS 字符串类名没转义 | 这个类名有没有被 Tailwind 扫描到 |

## 几个常见问题

不用一起升级到 Tailwind CSS 4。v5 同时支持 Tailwind CSS 3 和 4，想稳一点就先留在 3。

H5 / Web 构建可以继续用 Tailwind 官方插件。这里限制的是小程序构建。

JS 里的类名还会转，但前提是类名先被 Tailwind 扫描到。`weapp-tailwindcss` 不猜普通字符串，只处理 Tailwind 已确认的 classSet。
