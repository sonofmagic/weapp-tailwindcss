# 从 v4 迁移到 v5

`weapp-tailwindcss@5` 的生成模式默认开启，`tailwindcss@3` 和 `tailwindcss@4` 项目都会默认由 `weapp-tailwindcss` 接管 Tailwind CSS 样式生成。迁移时仍然必须先判断 Tailwind 主版本，因为两个版本的扫描配置和 CSS 入口写法不同。

## 核心变化

1. 生成模式默认开启，默认配置等价于 `{ mode: 'auto', target: 'weapp' }`。
2. `tailwindcss@3` 和 `tailwindcss@4` 项目在 `mode: 'auto'` 下都会由 `weapp-tailwindcss` 接管样式生成，失败时回退到旧的 CSS 后处理链路。
3. Vite 入口默认会移除 `@tailwindcss/vite`、`@tailwindcss/postcss` 和 `tailwindcss` PostCSS 插件，避免两套 Tailwind 生成链路同时运行。
4. CSS 入口必须是纯 `.css` 文件，不要把 Tailwind 入口或 `cssEntries` 指向 `scss`、`less`、`sass`。
5. Tailwind CSS v3 继续使用 `tailwind.config.js` 的 `content`，Tailwind CSS v4 继续使用 CSS 入口里的 `@source`。
6. JS / 模板类名仍按 `classSet` 精确命中转译，不会启发式转译普通字符串。

## 迁移路径选择

| 当前项目 | 推荐路径 | Tailwind 配置 | 官方 Tailwind 插件 | 改造成本 |
| --- | --- | --- | --- | --- |
| 已经使用 `tailwindcss@4` | 升级到 `weapp-tailwindcss@5`，使用默认生成模式 | CSS-first，使用 `@source` | 移除 `@tailwindcss/vite` / `@tailwindcss/postcss` | 需要整理 CSS 入口 |
| 仍在使用 `tailwindcss@3` | 升级到 `weapp-tailwindcss@5`，使用默认生成模式 | `tailwind.config.js` 的 `content` | 移除 `tailwindcss` PostCSS 插件 | 需要把 CSS 入口交给 `cssEntries` |

## 依赖升级

```bash
pnpm add -D weapp-tailwindcss@5 tailwindcss@4
```

如果项目暂时还使用 `tailwindcss@3`，推荐只升级 `weapp-tailwindcss`：

```bash
pnpm add -D weapp-tailwindcss@5 tailwindcss@3
```

`tailwindcss@4` 项目使用 CSS-first 入口和 `@source`，`tailwindcss@3` 项目继续使用 `tailwind.config.js` 的 `content`。两者都会默认走 v5 直出生成。

## Tailwind CSS v4 配置迁移

Tailwind CSS v4 项目迁移后只保留 `weapp-tailwindcss` 的构建器入口，让它成为唯一的 Tailwind CSS 生成入口。

### Vite / weapp-vite / uni-app Vite

移除 `@tailwindcss/vite` 和 `@tailwindcss/postcss`：

```ts title="vite.config.ts"
import path from 'node:path'
import { defineConfig } from 'vite'
import { WeappTailwindcss } from 'weapp-tailwindcss/vite'

export default defineConfig({
  plugins: [
    WeappTailwindcss({
      rem2rpx: true,
      cssEntries: [
        path.resolve(__dirname, 'src/main.css'),
      ],
    }),
  ],
})
```

### Taro Vite

```ts title="config/index.ts"
import path from 'node:path'
import { defineConfig } from '@tarojs/cli'
import { WeappTailwindcss } from 'weapp-tailwindcss/vite'

export default defineConfig({
  compiler: {
    type: 'vite',
    vitePlugins: [
      WeappTailwindcss({
        rem2rpx: true,
        cssEntries: [
          path.resolve(__dirname, '../src/app.css'),
        ],
      }),
    ],
  },
})
```

### Webpack

Webpack 项目继续注册 `weapp-tailwindcss/webpack`，但不需要显式写 `generator: true`：

```js title="vue.config.js"
const path = require('node:path')
const { WeappTailwindcss } = require('weapp-tailwindcss/webpack')

module.exports = {
  configureWebpack(config) {
    config.plugins.push(
      new WeappTailwindcss({
        rem2rpx: true,
        cssEntries: [
          path.resolve(__dirname, './src/app.css'),
        ],
      }),
    )
  },
}
```

Tailwind CSS v4：

```css title="src/main.css"
@import "tailwindcss";
@source "./**/*.{vue,js,ts,wxml,tsx,jsx}";
@source not "../dist";
@source not "../unpackage";
```

## Tailwind CSS v3 配置迁移

Tailwind CSS v3 项目不需要改成 CSS-first，但需要让 `weapp-tailwindcss` 成为 Tailwind 样式生成入口。迁移后移除原来的 `tailwindcss` PostCSS 插件，扫描范围仍然写在 `tailwind.config.js` 的 `content`。

```ts title="vite.config.ts"
import path from 'node:path'
import { defineConfig } from 'vite'
import { WeappTailwindcss } from 'weapp-tailwindcss/vite'

export default defineConfig({
  plugins: [
    WeappTailwindcss({
      rem2rpx: true,
      cssEntries: [
        path.resolve(__dirname, 'src/main.css'),
      ],
    }),
  ],
})
```

```css title="src/main.css"
@tailwind base;
@tailwind components;
@tailwind utilities;
```

```js title="tailwind.config.js"
export default {
  content: ['./src/**/*.{vue,js,ts,wxml,tsx,jsx}'],
  theme: {
    extend: {},
  },
}
```

## 兼容旧链路

如果项目暂时仍需要沿用“先由官方 Tailwind 插件生成浏览器 CSS，再由 weapp-tailwindcss 后处理”的链路，可以显式关闭生成模式：

```ts
WeappTailwindcss({
  generator: false,
})
```

如果希望 CI 中严格校验生成模式，不允许失败后回退，可以使用：

```ts
WeappTailwindcss({
  generator: {
    mode: 'force',
  },
})
```

## 检查清单

1. 移除项目中的 `@tailwindcss/vite`、`@tailwindcss/postcss` 或 `tailwindcss` PostCSS 插件注册。
2. 确认 `cssEntries` 指向纯 `.css` 入口文件。
3. Tailwind CSS v3 项目确认 `tailwind.config.js content` 覆盖模板和脚本文件。
4. Tailwind CSS v4 项目确认 `@source` 覆盖模板和脚本文件。
5. 使用 `twMerge`、`twJoin`、`cva`、`tv` 时，确认 `ignoreCallExpressionIdentifiers` 覆盖了实际调用名。
6. 验证基础工具类、任意值（含 `rpx`）和变体类都能在目标小程序端生效。

## 推荐验证命令

```bash
pnpm --filter weapp-tailwindcss test
pnpm build:apps
pnpm build:demo
```

业务项目中至少运行一次对应端构建，并检查生成的 `app.wxss` / `app.acss` / `app.ttss` 中是否已经输出小程序转义后的选择器。
