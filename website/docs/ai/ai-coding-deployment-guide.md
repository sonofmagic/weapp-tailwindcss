# AI 编程助手落地实施方案（决策与执行）

> 技术总监视角，围绕“选型→落地→规模化”，给出决策框架、价格要点、落地路线与执行清单。汇率假设 1 USD = ¥7.2，价格以 2025 Q1 公示为参考，落地前请再核实官网。

## 决策框架（先决条件与优先级）

1) 网络与合规：是否有稳定海外代理与合规豁免？若否，先国产（GLM/Qoder），海外白名单兜底。  
2) 团队治理：是否需要 SSO、审计、团队配额、账单中心？需要则选团队/企业版。  
3) 体验 vs 成本：体验极高 → Cursor/Claude；预算/报销友好 → GLM/Qoder；已有 OpenAI/Google 账单 → Codex/Gemini。  
4) 任务类型：日常补全用低价/国产；大段改造/评审才切高阶模型。  
5) 仓库敏感度：涉密仓库默认禁用出境；公共/开源可用海外。  
6) 工作流覆盖：若需 PR 审查、CI 生成测试/文档，优先选有官方插件/CLI/CI 集成的方案。

## 选型决策（简明树）

- 有稳定海外网络且能刷卡？  
  - 是 → 体验优先：Cursor；CLI/Agent 优先：Claude Code；已有 OpenAI/Google 账单：Codex/Gemini。  
  - 否 → 直接 GLM 或 Qoder。  
- 需 SSO/审计/团队配额？ → Cursor Business / Claude Teams / Qoder Team / GLM 企业版。  
- 代码隐私敏感？ → 国产/本地索引为主，海外仅用于脱敏仓库。  

## 成本估算（落地时要盯）

- 订阅固定成本：选最低能覆盖主力开发的档位，避免过度采购。  
- 超额与按量：海外模型设置硬上限与告警；国产利用刷新窗口批量执行。  
- 带宽成本：团队代理按“并发人数 * 2”带宽预留，避免高延迟放大调用成本。  
- 样例预算：  
  - 个人：GLM Pro ¥100 或 Qoder Pro ¥59 + Cursor Pro ¥144（体验补充）→ ¥159-244。  
  - 小团队 5 人：Cursor Business ~¥1,440；或 GLM Pro*5 ~¥500；或国产主力 + 2 席海外高阶约 ¥800-1,200。  
  - 大团队 30 人：20 人国产 Pro（¥2,000）+ 5 人海外高阶（¥1,000-3,000）+ 5 人 Business/Teams（¥1,400-1,800）≈ ¥4,400-6,800。  

## 落地路线图（公司多团队循序渐进）

### 阶段 0：基础准备（1 周）
- 网络与合规：确认海外代理出口、白名单域名；涉密仓库默认禁用出境模型。  
- 支付与账单：准备信用卡/虚拟卡（海外），支付宝/微信（国产）；设置账单上限与告警。  
- 工具基线：统一 IDE/CLI 版本，系统代理；本地索引策略排除 `.env`、密钥目录。  

### 阶段 1：试点（2-4 周，1-2 个低风险团队/仓库）
- 方案搭建：如 Cursor + GLM 兜底，开启团队版/SSO。  
- 落地动作：插件/CLI 登录；模型路由默认国产/低价，高阶按需；仓库忽略清单。  
- 观测指标：补全接受率、PR review 采纳率、平均/95 分位延迟、用量/超额、人工校对时间。  
- 成本防护：团队用量上限、个人上限，超额降级到低价/国产。  
- 培训：15 分钟速训（快捷键、提问模板、Do/Don’t），记录误用案例。  
- 回滚：代理或模型异常时，预设插件开关/路由降级。  

### 阶段 2：业务线扩展（4-8 周，同业务线其他团队）
- 模板复用：代理、账单中心、索引黑名单、CLAUDE.md 或工作区配置。  
- 分层路由：  
  - 基线：国产（GLM/Qoder）+ 本地索引。  
  - 提升：评审/复杂改造才切海外高质量模型，动作留痕。  
  - 阶梯限额：按团队等级设月/周上限，触顶自动降级。  
- 培训与守则：10-15 分钟速训，强调禁止上传敏感文件。  

### 阶段 3：全公司与 CI 集成（持续）
- PR/CI：PR 阶段触发 AI Review，限制 diff 大小/文件类型；CI 用低价模型生成测试/文档草稿。  
- 安全：CI 调用最小权限 token，日志到内部系统；跨境调用需白名单+审批。  
- 监控：月/周用量、超额、延迟、代理可用性；人均 AI 成本、单位 PR/提交成本。  
- SOP：  
  - PR 审查：小改用低价模型（Haiku/Flash/GLM Lite），大改/核心用高阶并标记来源。  
  - 测试生成：只出草稿，必须人工审阅；禁止覆盖现有测试。  
  - 文档/变更日志：草稿可由模型生成，发布前人工校对。  

### 阶段 4：优化与回溯（季度节奏）
- 成本优化：海外任务前置到订阅额度内，超出改低价；国产集中在刷新后执行。  
- 体验优化：依据延迟/接受率调整路由（如 Cursor 切 Composer，GLM 升级 Max）。  
- 合规复核：更新索引黑名单、出境名单，检查日志留存与审批流。  

### 阶段 5：多团队循序渐进推广（公司级）
- 波次推进：先 1-2 低风险团队试点 2-4 周，记录用量/延迟/命中率/人工校对，再扩展。  
- 角色分工：  
  - 中台/平台：统一代理、账号池、用量告警、模型路由、审计与报表。  
  - 安全/合规：敏感路径黑名单、出境策略、日志留存周期，审批海外模型。  
  - 财务/采购：海外卡或虚拟卡额度管理；国产发票/成本中心。  
- 路由分层：默认国产 + 本地索引；评审/复杂改造可申请海外高阶；阶梯限额触顶自动降级。  
- 指标复盘：补全接受率、PR 采纳率、误报/漏报；人均/单位 PR AI 成本；延迟与代理故障。  
- 风险缓释：涉密仓库禁出境；海外调用按目录/分支/成员白名单；推广在业务低峰期并有回滚方案。  

## 方案落地要点（按产品拆解）

### Cursor（海外，体验优先）
- 落地：下载 → 订阅 Pro/Business → 配置代理 → 选模型（Composer/Claude/GPT） → 开启 @Codebase 索引。  
- 团队：Business 开 SSO/配额/审计；限制 premium 消耗。  
- 风险：需稳定代理，代码出境；premium 约 500 次/月，峰值可能排队。  
- 建议：为核心仓库加 `.cursorignore`（忽略 `.env`、`secrets/`、`*.pem`、`node_modules`），禁自动上传二进制。  

### Claude Code / Codex / Gemini（海外订阅）
- 落地：官方插件/CLI 登录 → 设用量上限 → 默认低价模型，必要时手动切高阶。  
- 团队：Teams/Enterprise 开配额、审计、账单中心；跨境调用需审批。  
- 风险：超额按量昂贵；代理抖动会放大成本。  
- 建议：`CLAUDE.md`/工作区配置声明可访问/忽略路径；PR Review 设 diff 阈值与文件白名单。  

### GLM Coding Plan（国产，高刷新）
- 落地：实名 + 订阅 Pro/Max → 插件/CLI → 本地索引 → 按 5 小时刷新节奏批量执行。  
- 团队：集中采购 Pro/Max 或个人报销；设用量告警。  
- 风险：刷新是硬上限；外部 API 仍需出网策略。  
- 建议：批量任务安排在刷新后 0.5-2 小时；每日/每周用量看板提前调度。  

### Qoder（国产/混合）
- 落地：注册 → 选 Pro/Pro+/Team → 插件登录 → 国产优先，必要时启海外混合；开本地索引。  
- 团队：Team 版提供审计/成员管理；可设操作留痕与配额。  
- 风险：海外路由需代理；免费额度易触顶。  
- 建议：海外模型只在审批通过的仓库/分支启用；默认不开启文件上传。  

## 网络与合规硬性要求

- 海外模型必须走稳定代理，CLI/IDE 统一系统代理，节点优选美/日/新加坡。  
- 支付：海外用信用卡/虚拟卡并设硬上限；国产用支付宝/微信，便于报销与发票。  
- 数据：涉密仓库禁出境；敏感目录黑名单；操作/调用日志可追溯。
