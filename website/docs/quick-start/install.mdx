# Tailwind CSS 3.x 生成模式安装与配置

`weapp-tailwindcss@5` 默认使用生成模式。Tailwind CSS 3.x 项目继续使用 `tailwind.config.js` 的 `content` 扫描范围，但样式生成由 `WeappTailwindcss` 构建器插件接管，不再把 `tailwindcss` 注册到 PostCSS。

## 1. 安装依赖

```bash npm2yarn
npm i -D tailwindcss@3 weapp-tailwindcss
```

```bash npm2yarn
npx tailwindcss init
```

如果项目已经有 `postcss.config.js`，可以继续保留 `postcss-import`、`autoprefixer` 或框架内置插件等非 Tailwind 后处理插件；不要再添加 `tailwindcss` PostCSS 插件。

## 2. 配置扫描范围

```js title="tailwind.config.js"
/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [
    './public/index.html',
    './src/**/*.{html,js,ts,jsx,tsx,vue,wxml,axml,ttml,mpx}',
    '!./src/uni_modules/**/*',
    '!./node_modules/**/*',
    '!./dist/**/*',
    '!./unpackage/**/*',
  ],
  theme: {
    extend: {},
  },
}
```

`content` 应只覆盖业务源码。`uni_modules`、`node_modules`、`dist`、`unpackage`、文档和构建产物默认都应排除；如果必须扫描某个第三方包，请精确包含承载模板类名的文件。

## 3. 添加 CSS 入口

Tailwind CSS 3.x 入口仍使用 `@tailwind` 指令：

```css title="src/app.css"
@tailwind base;
@tailwind components;
@tailwind utilities;
```

这个文件需要在框架入口中引入，并在 `WeappTailwindcss` 的 `cssEntries` 中作为绝对路径传入。这样生成器、模板转译和 JS 转译会共享同一份 classSet。

## 4. 注册构建器插件

不同框架的注册位置不同，但原则一致：只注册 `WeappTailwindcss`，不要再注册 Tailwind 官方生成插件。

| 框架 | 构建入口 | 注册位置 | 文档 |
| --- | --- | --- | --- |
| uni-app Vue3 Vite | `weapp-tailwindcss/vite` | `vite.config.ts` 的 `plugins` | [uni-app cli vue3 vite](/docs/quick-start/frameworks/uni-app-vite) |
| Taro Webpack | `weapp-tailwindcss/webpack` | `config/index.ts` 的 `mini.webpackChain` | [Taro](/docs/quick-start/frameworks/taro) |
| Taro Vite | `weapp-tailwindcss/vite` | `config/index.ts` 的 `compiler.vitePlugins` | [Taro](/docs/quick-start/frameworks/taro) |
| weapp-vite 原生小程序 | `weapp-tailwindcss/vite` | `vite.config.ts` 的 `plugins` | [原生打包方案](/docs/quick-start/frameworks/native) |
| Mpx | `weapp-tailwindcss/webpack` | `mpx.config.js` / `vue.config.js` 的 `configureWebpack` | [Mpx](/docs/quick-start/frameworks/mpx) |

## 5. 旧项目迁移检查

| 旧配置 | 新处理方式 |
| --- | --- |
| `postinstall: "weapp-tw patch"` | 删除，v5 不需要手动 patch |
| `postcss.config.js` 中的 `tailwindcss` | 删除，由 `WeappTailwindcss` 生成 |
| 过宽的 `content: ['./src/**/*']` | 收窄到业务目录，并排除第三方和产物目录 |
| 没有 `cssEntries` | 在构建器插件中传入 Tailwind CSS 入口绝对路径 |
